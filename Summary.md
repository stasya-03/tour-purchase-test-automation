## Запланировано: 
- Создать проект для автоматизации проверки веб-приложения покупки тура 
- Написать UI-тесты с использованием Selenide и JUnit5 
- Настроить подключение к базе данных и SQL-проверки 
- Подключить Allure и Report Portal для формирования отчётов 
- Подготовить документацию: Plan.md, README.md, Report.md, Summary.md

## Реализовано: 
- UI-тесты для сценариев с APPROVED и DECLINED картами  
- SQL-запросы для проверки корректности записей в базе данных  
- Настроена очистка БД после выполнения тестов 
- Подключена генерация Allure-отчёта 
- Оформлены все требуемые документы 
- Настроен Gradle-запуск тестов

## Не удалось реализовать:
- Интеграция с Report Portal (демо-версия не позволяет создавать токены)

## Причины, по которым что-то не было реализовано
- Ограничения демо-версии Report Portal.

## Сработавшие риски
- Несовместимость версий драйверов браузера и Selenide.
- Длительный поиск локаторов
- Ошибки при локализации текста уведомлений об ошибке
